▸ defaults

Default Client/Server configs These values can be passed as an object parameter to both 'listen' and 'connect' methods.

'secretKey' can be populated with a 16+ character string to be used for password-encryption of the messages. This is no replacement to proper encryption, but provides sufficient hashing at a decent enough speed for non-critical scenarios

defaults
Example
{ 
  hostname: '0.0.0.0',
  port: 3000,
  transport: Kalm.transports.TCP,
  serial: Kalm.serials.JSON,
  secretKey: null,
  profile: Kalm.profiles.dynamic
}
▸ profiles

Kalm.profiles Predicates for executing message queues. 'tick' represents the maximum allowed wait time (ms) after an item is added before processing the queue. 'maxBytes' is the maximum bytes that can be contained in a queue. Queues that are about to exceed that value as the result of pushing a new message to it will be processed before-hand. This value includes Kalm frame overhead.

profiles
Example
['dynamic', 'heartbeat', 'threshold', 'realtime', 'manual', { tick: number|null, maxBytes: number|null }]
